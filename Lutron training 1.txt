--- quantum hub - power supply and processor ---

can communicate to up to 99 devices
support BACnet, DMX integration
control 512 switch legs - physical / digital (ie DALI)
shade motors also count as switch legs
only has 1 NIC / mac address -- 2 rj45 ports - one is a service port
works on multicast by default for networking --- broadcast , any device on the network can hear it -- not as secure as unicast -- in many cases, multicast is required

can track timezones / DST , etc -- cannot track moving holidays 

server based system - not a cloud system
UI - quantum Vue -- transfers all info to the processor + xfer to the database
once the db is setup - set up the local web page - server hosts the

server CAN BE just for config if on a laptop - after setup, information lives on the processor
server can also run permanently on an on-site hardware server, so user can continue to manage
sqlexpress
RabbitMQ service -- message queuing protocol
has enough memory to serve 7 days or so of information then overwrites itself -- if they dont have a server its difficult to pull historical logs

recommend a 500gb HD on the server

web API also supported -- list of partners / mfgs who can use the API (ex. Crestron)

--- T-Series / EcoSystem / DALI ---

DALI -- open source standard -- limit of 64 addresses

EcoSystem - propr. version based on early version of DALI -- also limit of 64 addresses - NOT specifically designed for tunable white

T-Series (Discontinued) = tunable white series (created from EcoSystem) -- adjust balance between cool and warm LED - 2ch --- 1 address for warm white, 1 address for cool white.  important bc DALI has a limit of addresses on a DALI BUS (OR LOOP) --- LIMITED TO 32 drivers / addresses

DALI panel can come in single or dual loop model
majority get 2

Originally EcoSystem was better because you get Sensor Feedback - etc

DALI version 2 makes BOTH EcoSystem and T-Series mostly obsolescent 
subsections - DT6 and DT8
DT6 achieves tunable white via 2 address --- EQUAL TO T-SERIES
DT8 uses 1 address for white tuning and color intensity -- can use 64 instead of 32 devices

*most things are compatible with athena*

TEST -- press and hold - test led start flashing
THEN press LOOP 1 --
-- press 1 time, FULL ON
-- press 2nd time, DIM
-- press 3rd time, Dimming up and down / flash
-- press 4th time, OFF
press and hold TEST again to stop test mode

Ethernet port = for standalone programming (if there is no processor)

Emergency contact functions the same way as before - but the indicator is different -- indicators flash red when in emergency mode (on loop 1 and loop 2 indicators at top)
SOLID RED --> SHORT
FLASHING RED --> IN EMERGENCY MODE
1 BLINK PER SEC / GREEN --> NORMAL


If temp light is flashing red and green - overheating issue

--- QS Link Cable ---

Plenum and non plenum - higher durability - open environment / air return or air supply area

Red and Black - 18 gauge

Red Black and Orange (orange cable one compatible with LEGACY) - 12 gauge - can go as far as 2000ft -- "did you buy 500 ft spools or 1000ft spools?" - if they used 4 spools on the floor theyre probably under 2000 but cutting it close ---- extra orange cable interfaces with the legacy panels -- typically only used for an emergency single

BOTH variations can come in 12, 16 or 18 gauge

can get non-Lutron cable but needs to follow the specs - can void warranty if the cable does not follow specs

1 PDU = 33 mA
when PDUs are added together, the count must be 0 or greater
QS link has a total limit of 100 total devices (*Wired Devices*) -- processor included -- does NOT include wired sensors


QS link power supply +8 PDUs - Seetouch -1 PDU
at the power supply units, terminal 2 (POWER SOURCE) is capped off / never connected

wired sensors also draw PDUs
