--- GRACE PERIOD ---
if the lights go off, you get a grace period to get the lights to go back to the previous level
**under advanced occupancy settings --> grace period should be turned on**
if you are going to use vacancy, you MUST have grace period on

this setting may NOT be set from the initial database you receive

grace period is used for OCCUPANCY to GO BACK TO THE PREVIOUS LEVEL, if not set, when the occ sensor is triggered again it will immediately go to the SET OCCUPIED LEVEL, (which is NOT the behavior you would typically want if waking the room back up when you are already sitting it)

default is 15 sec



WIRED OCC SENSOR will stay in test mode for 60 minutes**

-----  FIRMWARE UPDATES ----
note -- QS device firmware DOES NOT get automatically updated.  PROC and CCX firmware is automatically updated

QS devices can be upgraded via Tools -> Upgrade QS devices. (firmware)

you can etiher upgrade all devices with avialble updates, or you can select devices to manually upgrade.  sometimes you want to just select a specific device to upgrade because it takes SIGNIFICANT TIME to update, and you may have to leave site, or may not be working on that device currently.

when it updates the firmware, it pushes out all devices in the family type in parallel -- for example it will push out all the QSM firmware first, then the ESN firmware after. --- usually best to just pick a product family at a time to upgrade.  might take multiple hours to upgrade ALL devices

NOTE -- if you DO NOT PUSH THE FIRMWARE FULLY TO THE DEVICE, it may be STUCK IN BOOT MODE.  it can be recovered but this can cause issues -- does not automatically roll back to working firmware  -- you can sometimes recover by redoing the firmware update, but sometimes designer does not like units that are stuck in boot mode.

causes of QS Link noise
----
- too long of wire run for the gauge (12 AWG = 2000ft , 18 AWG = 500 ft  --- ALL the wire on the link must be below this number .. not just a single wire)
- mux and mux bar reversed / miswire
- loose connection (loose wire connection OR loose terminal block) -- Q: pigtailing wires (??) --- if a device is dropping on and off the link, it may indicate a loose connection near the device
- wires in proximity to parallel line voltage
- **if drain wire is connected to ground**, or is not connected through properly / not spliced togther or through properly
- not enough power / drawing too many PDUs / running low on PDUs -- devices may drop off the link and come back on
- backfeed power on the QS Link--- may cause devices to drop off the QS link because multiple devices detect incoming power and they try to negotiate back and forth
- common connected to earth ground or touching something that is shorting it to ground could cause the entire QS link to go down
