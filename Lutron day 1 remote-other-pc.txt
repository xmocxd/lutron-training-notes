--- ATHENA SYSTEM ARCHITECTURE ---


athena hubs sometimes referred to as processors, but technically the processor lives inside of the hub
wired processor = edge

ARF-2 - Athena Wireless Processor
Q-RF - Athena Gateway

QS-Link - wired low voltage communication link

Athena DIN panel - panels with power modules inside ---> centralized load controller - all the modules live inside
 
ESN = ****distributed**** load controllers - anywhere from 4 outputs .... to 128 switch legs it can control if it's ecosystem, etc.
distributed = scattered around the building, closer to where the loads are, so u dont have to wire everything back to the same home location
DMX Interface - is another type of distributed load controller

Wall Controls - all SPEAK QS

QSM - unique interface - 1) wired sensor inputs (including wired picos) - NON addressable devices, you have to tell what is on the inputs, they have to be landed on the correct terminal.  QS link also provides POWER to the devices.  -- 2) wireless sensor inputs

WE support Lutron shades on athena -- BUT they need power, they do not get power from the QS ink -- they require a separate share power supply - can be an individual or shared supply

ESN provides link power

INTERFACES -> AV interface - connect 3rd party equipment, can monitor and output commands / Contact Closure Interface - connect 3rd party equipment -- for example 3rd party shades, security system, etc.


BLUE LINE on diag - QS link
GREEN LINK - ethernet

inside the athena hub, there is a POE switch providing power and connectivity on ethernet

Touchscreen - can have access to 1 area, or multiple areas -- it is configurable
There is no programming per se of the touchscreen - it reads information from the processor

Athena Wireless Processor - dual radio:  clear connect type A (CCA) - it also has a CCX radio built into it - works with Ketra and AWNs--- it ONLY LISTENS CCA - does not talk CCA

also there is a CCX interface - not shown on the arch diagram --- looks the same as an athena wirelss processor, but does not support CCA

Software -- Lutron Dashboard & Lutron App / Lutron Designer
Dashboard - allows you to see alerts, system energy, system occupancy **history**, energy usage **history**
Lutron Designer - software we use to start up the system --- usually end users dont use, but if they go through facility mgr training, they can get access
(need mylutron account to get to all software)


Working on an option to have a local server for athena that is not cloud connected / cloud required -- still in early stages



if you press a control, it sends the message ***all the way back to the processor***, then the PROCESSOR tells the device to react to the button press -- dispatches

EVERYTHING goes through the processor, ESNs/DIN modules (LOAD CONTROLLER) do not directly communicate to controls / sensors, they only get their commands from the processor and the load controller then manipulates

isolated system - 16 processors per system -- systems cannot talk to systems
